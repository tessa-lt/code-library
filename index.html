Code Library

It is recommended that you use some type of browser reset to remove default margins and padding. Elements such as list and table items may appear inconsistent if browser styles are not reset.

Browser Compatibility

  • Full support: modern webkit, Firefox, IE9+ (not yet tested in Firefox 3)
  • Good enough support: IE8 will render all layouts properly, but will be missing CSS3 gradients, box-shadows, and border-radii
  • Not-so-great: IE7 will not render layout or horizontal form elements correctly. Tabbed navigation will be usable, but current tab will have a bottom border.

Layout

Not supported in IE6/7

Since I'm not using floats, these should be self-clearing, provided all columns add up to 100% (or 99%)

$('a[rel=tooltip]').mouseover(function(e) {
    var tip = $(this).attr('title');

    $('#tooltip').css('top', e.pageY + 10 );
    $('#tooltip').css('left', e.pageX + 20 );

    $('#tooltip').css('left', e.pageX + 20 );
     
}).mouseout(function() {
 
    $(this).attr('title',$('.tipBody').html());
    $(this).children('div#tooltip').remove();
     
});

$(".tabcontainer > ul > li  a").click(function () {
